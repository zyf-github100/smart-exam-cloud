# Smart Exam Cloud PRD（产品需求文档）

## 1. 文档目的

本文件用于定义智能考试云平台的业务目标、角色场景、功能需求、业务规则与验收标准，供产品、开发、测试协同执行。

适用范围：

- 网关、认证、用户、题库、考试、判卷、分析、前端控制台全模块。

## 2. 产品背景与目标

### 2.1 背景

传统考试流程链路长、人工成本高、数据利用率低。本项目提供微服务化考试基础平台，支持从题库建设到成绩分析的闭环。

### 2.2 核心目标

- 提供可运行的在线考试主链路：出题、组卷、开考、答题、交卷、判卷、报表。
- 统一接入层（Gateway）与认证机制（JWT）。
- 通过 Redis 与消息队列提升并发场景下的稳定性与解耦能力。
- 为后续扩展真实判题引擎、监考能力、精细化权限体系预留架构空间。

## 3. 角色定义与权限边界

角色：

- `ADMIN`：系统管理与平台运维管理角色。
- `TEACHER`：题库管理、组卷、开考、判卷。
- `STUDENT`：参加考试、提交答案。

权限边界（当前实现）：

- 网关负责登录态校验与身份透传。
- `admin-service` 已提供角色-权限矩阵管理与审计日志能力。
- 业务服务侧（非 admin 域）细粒度 RBAC 仍需增强。

## 4. 业务范围

### 4.1 In Scope（本期范围）

- 账号登录与 JWT 鉴权。
- 用户资料查询。
- 题目管理与试卷管理。
- 考试创建、会话启动、答题保存、交卷。
- 判卷任务生成、人工评分、成绩发布。
- 分数分布、题目正确率 TopN 报表。

### 4.2 Out of Scope（暂不纳入）

- 全域（所有业务服务）细粒度 RBAC 权限中心一次性落地。
- 强一致的事务消息体系。
- 真实自动判题策略与主观题 AI 判卷。
- 严格防作弊能力（切屏、设备指纹、风险评分）的完整实现。

## 5. 业务流程（端到端）

### 5.1 流程 A：题库建设

1. 教师创建题目。
2. 系统校验并入库题目。
3. 教师查询题目列表和详情。

### 5.2 流程 B：组卷

1. 教师提交试卷信息与题目清单。
2. 系统校验题目是否存在。
3. 系统计算总分并保存试卷与题目关联。

### 5.3 流程 C：开考与作答

1. 教师创建考试（时间窗、防作弊等级）。
2. 学生开始考试，系统创建或复用进行中会话。
3. 学生多次保存答案。
4. 学生提交试卷，系统将会话置为已提交并发布交卷事件。

### 5.4 流程 D：判卷

1. 判卷服务消费交卷事件。
2. 生成判卷任务并计算客观分。
3. 若需人工评分，教师提交主观分。
4. 发布成绩事件。

### 5.5 流程 E：分析报表

1. 分析服务消费成绩事件并沉淀快照。
2. 前端查询分数分布与题目正确率报表。

## 6. 功能需求清单

### 6.1 网关模块（gateway-service）

- `FR-GW-001`：提供统一入口，按路径路由到各业务服务。
- `FR-GW-002`：白名单路径支持免鉴权（登录与健康检查）。
- `FR-GW-003`：非白名单请求必须校验 Bearer Token。
- `FR-GW-004`：鉴权成功后透传 `X-User-Id`、`X-Role`。
- `FR-GW-005`：鉴权失败返回统一错误结构（40100）。

### 6.2 认证模块（auth-service）

- `FR-AUTH-001`：支持用户名密码登录。
- `FR-AUTH-002`：登录成功返回 token、过期时间、用户摘要信息。
- `FR-AUTH-003`：登录请求支持短窗口防重，避免重复提交。
- `FR-AUTH-004`：支持演示账号自动补齐（admin/teacher1/stu1）。
- `FR-AUTH-005`：提供注销接口（无状态退出）。

### 6.3 用户模块（user-service）

- `FR-USER-001`：查询当前登录用户信息 `/users/me`。
- `FR-USER-002`：按 ID 查询用户详情。
- `FR-USER-003`：查询用户列表。
- `FR-USER-004`：用户详情和列表使用 Redis 缓存加速。

### 6.4 题库模块（question-service）

- `FR-Q-001`：创建题目，支持题型、难度、答案、选项、解析。
- `FR-Q-002`：查询题目列表与单题详情。
- `FR-Q-003`：创建试卷并保存题目分值及顺序。
- `FR-Q-004`：创建试卷前校验题目存在性。
- `FR-Q-005`：创建题目和试卷支持幂等防重。

### 6.5 考试模块（exam-service）

- `FR-EXAM-001`：创建考试，要求 `startTime < endTime`。
- `FR-EXAM-002`：考生在时间窗内可开始考试。
- `FR-EXAM-003`：同一考生同一考试存在进行中会话时应复用。
- `FR-EXAM-004`：保存答案仅允许会话所属考生执行。
- `FR-EXAM-005`：仅进行中会话可提交。
- `FR-EXAM-006`：提交动作需防重并发布 `exam.submitted` 事件。

### 6.6 判卷模块（grading-service）

- `FR-GRADING-001`：监听交卷事件并创建判卷任务。
- `FR-GRADING-002`：事件消费需支持幂等去重。
- `FR-GRADING-003`：支持按状态查询判卷任务。
- `FR-GRADING-004`：人工评分仅允许 `MANUAL_REQUIRED` 状态任务。
- `FR-GRADING-005`：人工评分后更新总分并发布 `score.published` 事件。
- `FR-GRADING-006`：`exam.submitted` 消费失败需进入重试队列，超过重试上限进入 DLQ。

### 6.7 分析模块（analysis-service）

- `FR-ANALYSIS-001`：监听成绩事件并按会话维度 Upsert 成绩快照。
- `FR-ANALYSIS-002`：提供分数分布报表接口。
- `FR-ANALYSIS-003`：提供题目正确率 TopN 报表接口。
- `FR-ANALYSIS-004`：报表接口支持缓存并在成绩更新后失效。
- `FR-ANALYSIS-005`：`score.published` 消费失败需进入重试队列，超过重试上限进入 DLQ。

### 6.8 前端联调模块（smart-exam-web）

- `FR-WEB-001`：支持登录、token 持久化、API Base 动态切换。
- `FR-WEB-002`：按标签页覆盖全链路调用场景。
- `FR-WEB-003`：支持判卷操作与报表图表展示。

### 6.9 管理员模块（admin-service）

- `FR-ADMIN-001`：提供管理员总览接口，输出用户、考试、阅卷、审计核心指标。
- `FR-ADMIN-002`：支持用户目录检索与分页查询。
- `FR-ADMIN-003`：支持管理员变更用户状态（启用/停用）并记录审计日志。
- `FR-ADMIN-004`：支持管理员调整用户角色并记录审计日志。
- `FR-ADMIN-005`：支持管理员重置用户密码并记录审计日志。
- `FR-ADMIN-006`：支持角色权限矩阵读取与更新。
- `FR-ADMIN-007`：支持系统配置读取与写入。
- `FR-ADMIN-008`：支持按条件检索管理员审计日志。
- `FR-ADMIN-009`：管理员写操作需具备短窗口防重能力（避免重复提交）。

## 7. 核心业务规则

### 7.1 题目与试卷

- 题目难度限定 `1..5`。
- 试卷总分由题目分值求和得出。
- 试卷题目顺序需可控，且同一试卷内顺序唯一。

### 7.2 考试与会话

- 考试开始时间必须早于结束时间。
- 只允许在考试时间窗内开始考试。
- 会话状态机：`IN_PROGRESS -> SUBMITTED`。

### 7.3 判卷

- 每个会话仅能产生一条有效判卷任务。
- 人工评分需记录评分人。
- 总分 = 客观分 + 主观分。

### 7.4 报表

- 分数分布按固定分桶输出。
- TopN 参数限制在合理区间（1..50）。

## 8. 非功能需求（产品视角）

- 安全：未登录请求不可访问核心业务接口。
- 可用性：重复请求不应导致重复业务结果。
- 一致性：异步链路应满足最终一致。
- 性能：查询接口应具备缓存能力。
- 可运营：关键流程可通过日志和健康检查定位问题。

## 9. 验收标准

- `AC-001`：完成登录、出题、组卷、开考、作答、交卷、判卷、报表的完整演示。
- `AC-002`：重复提交登录/创建/评分/交卷可被识别并返回冲突错误。
- `AC-003`：重复消费消息不会产生重复判卷任务或重复成绩快照。
- `AC-004`：报表结果可随评分结果更新。
- `AC-005`：网关统一鉴权与路由行为符合定义。
- `AC-006`：MQ 消费异常时可观察到“重试后成功”或“进入 DLQ”的可追踪结果。

- MQ 可靠投递与消费失败治理（重试、DLQ）[已完成，2026-03-04]。

- 考试状态自动流转任务。[已完成，2026-03-04]

- 全链路审计日志。
